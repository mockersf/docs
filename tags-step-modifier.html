Running in a Private Network

You may have a private cluster only reachable by special workers running on-premises. To run a step against those workers, just provide a matching tag:

plan:
- get: my-repo
- put: my-site
  tags: [private]
  params: path: my-repo
- task: acceptance-tests
  tags: [private]
  file: my-repo/ci/atteptance.yml

1.6.1.11 tags step modifier

Any step can be directed at a pool of workers for a given set of tags, by adding the tags attribute to it.

tags: [string]

Optional. Default []. The tags by which to match workers.

For example, if [a, b] is specified, only workers advertising the a and b tags (or any others) will be used for running the step.
